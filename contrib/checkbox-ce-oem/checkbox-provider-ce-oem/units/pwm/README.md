# PWM Tests

The PWM tests discover PWM outputs from `/sys/kernel/debug/pwm` and generate one
test job for each eligible PWM output.

## Common setup

Enable the PWM tests with the `has_pwm` manifest entry:

```ini
[manifest]
has_pwm = true
```

To find stable PWM controller names and PWM names on the target device, run:

```bash
sudo cat /sys/kernel/debug/pwm
```

Use the controller name from the header line, for example
`platform/10048000.pwm`, and the PWM output name, for example `pwm-0`.
Do not use `/sys/class/pwm/pwmchipN` names in launcher variables because
`pwmchipN` can change after reboot.

## `ce-oem-pwm/check`

This job lists the current PWM chip to PWM output mapping and reports the total
PWM chip count and PWM device count. When `PWM_DEV_TOTAL_NUM` is provided, it
also verifies that the discovered PWM device count matches the expected value.

### Optional environment variables

Use these variables in the launcher `[environment]` section when needed:

| Variable            | Description                      | Default |
|---------------------|----------------------------------|---------|
| `PWM_DEV_TOTAL_NUM` | Expected total PWM device count. | Not set |

### Example launcher environment

```ini
[environment]
PWM_DEV_TOTAL_NUM = 6
```

If `PWM_DEV_TOTAL_NUM` is not set, the job only reports the discovered mapping
and totals.

## `ce-oem-pwm/sysfs-validation`

This template job validates PWM sysfs controls for each PWM output generated by
the `ce-oem-pwm/pwm-list` resource job.

### Optional environment variables

Use these variables in the launcher `[environment]` section when needed:

| Variable              | Description                                           | Default   |
|-----------------------|-------------------------------------------------------|-----------|
| `PWM_IGNORE`          | Comma-separated controller or consumer names to skip. | Not set   |
| `PWM_ALLOW`           | Comma-separated active consumers or controllers to include. | Not set |
| `PWM_TEST_PERIOD`     | Period value in nanoseconds for the test.             | `1000000` |
| `PWM_TEST_DUTY_CYCLE` | Duty-cycle value in nanoseconds for the test.         | `500000`  |

### Example launcher environment

```ini
[environment]
PWM_IGNORE = platform/208c000.pwm,platform/32e0000.pwm
PWM_ALLOW = pwm-fan,platform/32a0000.pwm
PWM_TEST_PERIOD = 1000000
PWM_TEST_DUTY_CYCLE = 500000
```

`PWM_IGNORE` and `PWM_ALLOW` accept multiple values separated by commas. Each
value can be either:

- a stable controller name, for example `platform/10048000.pwm`
- a controller basename, for example `10048000.pwm`
- a consumer name, for example `pwm-fan`

`PWM_IGNORE` takes priority over `PWM_ALLOW` when both match the same PWM
output.
